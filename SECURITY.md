# Security policy

This repository backs a single static marketing site, uke-o-ono.com. There is
no user data, no login, and no server-side application code, so the security
surface is small. Reports are still welcome.

## Reporting a vulnerability

Please report security issues privately rather than opening a public issue:

- Use GitHub's [private vulnerability reporting](https://github.com/laazyj/ukeoono.com/security/advisories/new)
  ("Report a vulnerability" on the Security tab), or
- email the address listed on [uke-o-ono.com](https://uke-o-ono.com).

Include enough detail to reproduce the issue. As this is a personal project,
responses are best effort, but genuine reports will be looked at.

## Scope

In scope: this repository's source code and the configuration that deploys
uke-o-ono.com.

Out of scope: the third-party platforms the site relies on (AWS, Cloudflare,
GitHub, Instagram). Report issues with those to the relevant vendor.
